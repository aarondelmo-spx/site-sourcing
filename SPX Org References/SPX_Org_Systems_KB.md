## Part 2: Internal Systems

### FMS (Fleet Management System)
- Type: In-house platform
- Used by: Control tower, fleet ops, hub coordinators, LM ops, FM ops, sorting hub staff
- Functions: GPS/telemetry tracking for linehaul trucks, fleet management (driver/truck assignment, trip records), rider/driver management (onboarding, performance, incentives), sorting hub operations (scan tracking, parcel routing), allocation visibility (ground ops check allocation results here)
- Note: Single source of truth for daily operations. Most ops-facing roles interact with FMS daily.

### SeaTalk
- Type: Internal messaging platform (company-wide)
- Used by: Everyone
- Functions: Chat, group messaging, bot integration, announcements
- Note: Primary internal comms. SMART bots deploy here.

### Google Workspace
- Type: Productivity suite
- Used by: Everyone
- Functions: Google Sheets (default tool for tracking, reporting, manual processes), Google Docs (SOPs, documentation), Google Slides (presentations, reporting), Google Drive (file storage)
- Note: Google Sheets is effectively the system of record for processes without a dedicated platform, including: finance ops (invoicing, COD, incentives), claims processing, network expansion site tracking, scheduling validation, vendor management, budget tracking, KPI reporting.

### SMART (SEA Multi-Agent Realisation Technology)
- Type: No-code/low-code AI agent platform (Sea Group internal)
- Used by: Teams building AI agents
- Functions: Build and deploy AI agents with single-agent, multi-agent, and workflow architectures
- Available LLMs: GPT-4o, GPT-5.x, Qwen (good for Tagalog/Taglish), Claude, DeepSeek, Compass (in-house)
- Official tools: Data Service (Hive/SQL), Text2SQL, Confluence, Jira, Google Docs, Google Sheets, Web Crawler, Gmail, Google Calendar
- Deployment channels: SeaTalk bot, Live App, API, Scheduled Trigger
- Note: Mandated platform for AI use cases.

### Hive (Data Warehouse)
- Type: Data warehouse accessed via Data Service API
- Used by: Data analysts, BI, SMART agents
- Functions: Operational data — delivery volumes, fleet performance, financial data, rider metrics, hub throughput
- Note: Most reporting and analytics pull from Hive. SMART agents query via Data Service or Text2SQL tools.

### Confluence
- Type: Wiki/documentation
- Used by: All teams
- Note: SMART can ingest Confluence as KB and has a real-time Confluence tool.

### Jira
- Type: Ticketing/project management
- Used by: Tech teams, some ops teams
- Note: SMART has an official Jira tool.

### HRIS
- Type: HR information system
- Used by: HR, people managers
- Functions: Attendance, HC data, leave management, hiring pipeline

### Manual Processes (Google Sheets / Docs, no dedicated system)
When a JD mentions "reporting," "tracking," "reconciliation," or "processing" without specifying a system, it is almost certainly done in Google Sheets. Key manual processes include:
- Invoice processing, COD reconciliation, incentive payments (Finance ops)
- Claims processing — lost & found, recovery, compensation (Claims team)
- Network expansion site tracking — individualized per PM, no standard format (Network expansion)
- Scheduling and billing validation (Scheduling team)
- Vendor audit and tender management (Vendor management)
- Budget tracking and reforecasting (Multiple functions)
- KPI reporting — typically Sheets → Slides (Most functions)
